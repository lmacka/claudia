claudia — wireframe map

Click through any flow. Notes in italic are design intent, not literal copy. Every page uses the dark sage theme by default; the settings page shows the picker (blush / sage / lavender / amber / high-contrast).

Adult mode (Liam's deploy)

Single user. No /admin. Library is editable.

  1. Setup wizard (3 stages):
    • /setup/1 — name + DOB
    • /setup/2 — ingest documents, claudia auto-populates a profile draft
    • /setup/3 — recap + offer to add more, ready
  2. / — chat list + mood log
  3. /session/<id> — chat (mood at start AND end, full tools)
  4. /library — documents
  5. /people — relationship map
  6. /report — therapist handover PDF
  7. /session/<id>/review — memory review (button-driven, no internals shown)
  8. /settings — theme + prompt overrides

Kid mode (Jasper's deploy)

Two roles in one Helm release. Kid sees clean chat. Parent (deployer) sees /admin.

  1. Parent setup wizard (3 stages):
    • /setup/1 — kid's name + DOB + parent password
    • /setup/2 — drop docs about kid, claudia drafts a profile, parent reviews
    • /setup/3 — recap, generate break-glass envelope, hand-off URL
  2. / — kid's first ever login (lighter banner, sets passphrase)
  3. / — kid chat (action buttons, mood at start+end, persistent crisis footer that doesn't end the convo)
  4. /admin — parent dashboard (tile layout)
  5. /admin/review — parent Sonnet meta-audit
  6. /admin/library — parent manages library (same UI)
  7. /admin/break-glass — parent emergency-decrypt + passphrase reset

Changes from v1

  • Dark base with 5 theme options (sage default). Settings page picker.
  • Setup is 3 stages, not one mega-form. Stage 2 is "drop docs, claudia drafts a profile" — the wizard does work for the parent.
  • DOB stored, age computed.
  • Mood collected at session start AND end, not just end.
  • Memory-review uses plain language (no md filenames in the user-facing copy).
  • Kid first-chat banner trimmed from 4 bullets to 3, warmer.
  • Kid chat: "your parents" not "your dad". Hotlines show but conversation continues.
  • Admin home is tile layout, not a list.
  • Both chats: "+ attach" button + paste-from-clipboard for screenshots and docs.
  • Kid attachments: OCR via Sonnet vision → text saved to chat → original image discarded. Detected names cross-checked against the shared /people store; unknown names trigger an inline "remember?" prompt that adds to the same store.
  • Single people store shared between kid and parent (parent manages at /admin/people). Classmate names aren't confidential — parents already know who their kid hangs around with.
